# NAVY_BLUE Revenue Deck

副業映像制作の「案件・収益・入金」を1画面で管理するダッシュボード。
ネイビー／ブルー基調（＝管理・精度・信頼の色）。ビルド不要のバニラ HTML / CSS / JS。

Claude Code で作成（Navy Workdeck とは別プロジェクト）。

---

## できること

### Phase 1 — 案件管理
- 案件の登録・編集・削除（クライアント名 / 種別 / 金額 / 受注日 / 納期 / 請求日 / 入金日 / メモ）
- 6段階のステータスワークフロー
  `商談中 → 受注 → 制作中 → 納品済み → 請求済み → 入金済み`
- 「次へ ▶」ボタンでワンタップでステータスを進める（受注日・請求日・入金日を自動補完）
- 今月の入金・年間累計・未入金（税込）・目標達成率のサマリー
- 納期が近い案件 TOP3（超過アラート付き）

### Phase 2 — 可視化・目標
- 月別入金推移グラフ（当年12か月、当月を強調）
- 月間目標の設定と進捗バー（達成で色変化）

### Phase 3 — 請求書
- 案件から請求書プレビューを生成（税込・消費税10%・支払期限=請求日+30日）
- ブラウザの「印刷 / PDF保存」でそのまま PDF 化（請求書だけを印刷する CSS 済み）

### その他
- 案件の CSV 出力（Excel 文字化け防止の BOM 付き UTF-8 / CSVインジェクション対策済み）
- 月次サマリーを Markdown でクリップボードへコピー（Obsidian 貼り付け用）

## 使い方

1. `index.html` をブラウザで開くだけ（サーバー不要）。
2. 右上の「＋」でサンプル案件を投入してお試し可能。
3. データはこのブラウザ内（localStorage）に保存されます。

ローカルで配信したい場合:

```
python3 -m http.server 4188 --directory .
# → http://localhost:4188
```

---

## データ保存とクラウド同期

- 既定では **このブラウザ内（localStorage）** に保存。設定不要で動きます。
- 任意で **Supabase** を設定すると、Mac・iPad・会社PC で同じデータを共有できます。

### Supabase 同期の手順（任意）

1. Supabase の SQL Editor で `supabase-schema.sql` を実行（`Success. No rows returned` で成功）。
2. アプリの「CLOUD SYNC」を開き、`Supabase URL` と `anon public key`（公開用キー）を入力して保存。
3. ログイン用メールを入れて「ログインリンク送信」→ メールのリンクからログイン。
4. 「ローカル分を同期」で、これまでの案件をクラウドへ送れます。

> anon public key は公開前提の鍵なので localStorage 保存で問題ありません。
> 行レベルセキュリティ（RLS）で「自分の行だけ」読み書きできるようになっています。
> サービスロールキーは絶対に入れないでください。

## 設計メモ

- **売上＝税抜**で集計（会計上の売上の定義）。一方 **未入金＝税込**で表示（実際に回収する資金繰りの額）。
- 「入金済み」案件は入金日（`paidAt`）で月次に計上。入金日が無い場合は登録時に確定させ、後の編集で売上月がずれないようにしている。
- 日付はすべてローカルタイムで処理（UTC変換による1日ズレを回避）。
- クラウド同期の競合は「編集時刻が新しい方を採用」。DBトリガが上書きする `updated_at` と区別するため、編集時刻は `client_updated_at` に保持。
